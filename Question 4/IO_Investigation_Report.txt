IO Investigation Report

Date: 03 July 2026

Commands Executed:

1. lsof | head
   Result:
   - The command could not be executed because the lsof utility was not installed.
   - Output:
     bash: lsof: command not found

2. ls -l /proc/$$/fd
   Purpose:
   - Displayed the file descriptors currently used by the running shell.
   Observation:
   - File descriptor 0 -> Standard Input (stdin)
   - File descriptor 1 -> Standard Output (stdout)
   - File descriptor 2 -> Standard Error (stderr)
   - File descriptor 255 -> Current shell terminal

3. echo "Hello Linux" > output.txt
   Purpose:
   - Redirected standard output to the file output.txt.
   Result:
   - The file output.txt was created containing the text:
     Hello Linux

4. ls unknownfile 2> error.txt
   Purpose:
   - Redirected the error message to error.txt.
   Result:
   - The error generated by the ls command was stored in error.txt.

5. ls > files.txt 2> errors.txt
   Purpose:
   - Redirected standard output to files.txt.
   - Redirected standard error to errors.txt.
   Result:
   - Directory listing was saved in files.txt.
   - Since there were no errors, errors.txt remained empty.

6. ulimit -a
   Purpose:
   - Displayed the current resource limits for the shell.
   Information displayed included:
   - Maximum open files
   - Stack size
   - Virtual memory
   - CPU time
   - File size
   - Maximum user processes
   - Other system resource limits

Summary:
The investigation demonstrated Linux input/output redirection, file descriptor management, and process resource limits. Although the lsof command was unavailable in the Cloud Shell environment, the /proc/$$/fd directory successfully displayed the active file descriptors of the current shell process.
